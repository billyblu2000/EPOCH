# Writing Workspace — 协作写作空间详细设计

## 一、从 Planning 到 Writing 的衔接

当 Planning Pipeline 完成后（至少完成因果一纪），系统自动执行：

1. **创建所有章节**：根据《因果录》自动创建全部章节条目。
2. **填充章节元信息**：每章预填章节名称 + 1-2 句话核心剧情描述。
3. **正文留空**：等待用户进入 Writing Workspace 逐章创作。

**前置条件**：进入某章的写作前，该章必须已完成因果二纪（分镜生成）。若未完成，系统引导用户先完成该章分镜。

---

## 二、沉浸式 UI 三栏布局

```
┌──────────────┬──────────────────────────────┬──────────────┐
│              │      章节名称 + 1-2句总结      │              │
│   左翼        │                              │   右翼        │
│   命理镜      │        中轴·创作区域           │   众生相      │
│  （分镜流）    │       （纯净写作区）           │ （人物侧边栏） │
│              │                              │              │
│              │                              │              │
│              │                              │              │
└──────────────┴──────────────────────────────┴──────────────┘
```

### 中轴：创作区域
- **纯净写作区**：富文本编辑器，用户在此撰写正文。
- **上方常驻**：当前章节的名称 + 《因果录》中的 1-2 句核心剧情总结。
- 这是用户 90% 时间注视的区域，保持极简。

### 左翼：命理镜（分镜流）
- 显示当前章节的**完整分镜列表**（来自《命理镜》）。
- 每个分镜卡片展示：镜头序号、登场人物、场景、事件摘要。
- **交互**：点击某个分镜卡片，AI 自动将写作上下文定位至该分镜，辅助生成时优先参考该分镜描述。
- 已完成的分镜可标记为"已写"，便于追踪进度。

### 右翼：众生相（人物侧边栏）
- **动态显示**：自动识别当前章节的登场人物（基于《命理镜》中的人物信息）。
- 每个人物卡片展示：姓名、性格倾向、口癖/行为癖好、与本章相关的关系。
- 点击人物卡片可展开查看完整人物设定。
- **作用**：写对话时随时参考人物口吻，确保一致性。

## 三、交互语法：`#` 与 `@` 引擎

### `#` 续写触发

用户在正文中输入 `#` 后跟具体意图，按下回车，触发 AI 生成。

> **注意**：写作区域为纯文本/富文本编辑器，**不支持 Markdown 语法**。`#` 在本编辑器中仅用于触发 AI 续写指令，不会被解析为标题。

#### 语法格式
```
# [自然语言指令]
```

#### 示例
```
# 详细描写两人对峙时的张力
# 描写夜晚森林中的诡异氛围
# 写一段白璃与龙清瑶的激烈争吵
```

#### AI 生成时的 Prompt 自动打包逻辑
当用户按下回车触发生成时，系统自动将以下上下文注入 AI Prompt：

| 上下文来源 | 内容 |
|-----------|------|
| 当前分镜描述 | 左翼命理镜中当前激活的分镜详细描述 |
| 人物设定 | 右翼众生相中当前章节登场人物的设定 |
| 上文内容 | 创作区中光标位置之前的约 500 字正文 |
| 用户指令 | `#` 后面的自然语言内容 |
| 全局设定 | 必要时引用《创世书》《神世法》中的关键约束 |

#### 状态锁定
- AI 生成期间，创作区域进入**灰度锁定状态**。
- 用户无法修改内容，防止逻辑流被中途破坏。
- 生成完成后解锁，用户可编辑生成的内容。

---

### `@` 精准注入

在 `#` 指令内部，用户可以使用 `@` 语法精准注入实体和描写维度。

#### 语法格式
```
# ... @[实体名/维度/字数] ...
```

#### 三类 `@` 标签

| 类型 | 示例 | 说明 |
|------|------|------|
| **实体标签** | `@白璃`、`@玄甲军`、`@青云宗` | 引用《众生相》或《神世法》中的具体实体，将其设定注入 Prompt |
| **描写维度标签** | `@外貌`、`@场景`、`@动作`、`@心理` | 提示 AI 切换描写侧重点 |
| **长度标签** | `@100`、`@200`、`@500` | 提示 AI 生成约指定字数的内容 |

#### 交互流程
1. 用户输入 `@`。
2. 弹出**下拉选择框**：
   - 来源一：《众生相》中的人物名。
   - 来源二：《神世法》中的场景/势力/实体名。
   - 来源三：预设描写维度（外貌、场景、动作、心理等）。
   - 来源四：预设字数选项（100、200、300、500 等）。
3. 用户选择或手动输入。
4. 可在一条 `#` 指令中使用多个 `@` 标签。

#### 综合示例
```
# @白璃 @心理 此时对龙清瑶的背叛感到极度愤怒但表面隐忍 @200
```
解读：
- 注入白璃的人物设定。
- 侧重心理描写。
- 用户意图：愤怒但隐忍。
- 目标字数约 200 字。

## 四、章节导航

- 左侧（或顶部）提供**章节列表**，可快速切换章节。
- 每章显示写作进度（已写字数 / 目标字数）。
- 未完成分镜的章节标记提示，点击跳转至因果二纪补全。
