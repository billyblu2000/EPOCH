# Planning Pipeline — 六大纪元详细设计

## 通用模式

每个纪元严格遵循三段式流程：

1. **结构化输入**：用户通过表单/卡片/时间轴等结构化 UI 填写信息（AI 可预填，用户可修改/增删）。
2. **AI 联想生成**：AI 基于结构化输入 + 前序纪元产出物，生成数千至上万字的非结构化纯文本描述。不满意可重新生成。
3. **文本化定稿**：用户在生成结果上进行人工调整（编辑文本），确认无误后锁定为该纪元的产出物，进入下一纪元。

### 重新生成策略

- **重新生成会覆盖当前内容**：用户点击"重新生成"后，之前的 AI 生成内容及手工编辑将被新生成内容替换，不保留历史版本。
- MVP 阶段不提供版本历史或 diff 对比功能。

### 纪元回退策略（MVP 阶段）

- MVP 阶段**不实现**纪元回退/跨纪元编辑功能。用户完成并锁定某纪元后，不可返回修改。
- 此功能列入后续迭代计划。
- **代码层面预留**：各纪元的 `status` 字段使用 `draft | finalized` 状态机，产出物（`artifact_text`）与结构化数据分开存储，确保未来实现回退（解锁 status → 重新编辑 → 级联提示下游纪元重新审视）时无需大规模重构。

### 产出物双重存储

每个纪元同时保留两份数据：
1. **结构化数据**：用户在表单/卡片/时间轴中填写的原始结构化内容（存储在各纪元对应的结构化字段/子表中）。
2. **非结构化产出物文本**（`artifact_text`）：AI 基于结构化数据生成的完整叙述文本。

两份数据均持久化保存。AI 生成后续纪元的 Prompt 时，可根据需要灵活选择引用结构化数据（精简、精确）或非结构化文本（完整叙述），具体的 Prompt 工程策略在接入真实 LLM 时调整优化。

## 纪元一：创世纪 (Genesis) —— 核心逻辑建模

### 目标
构建小说的核心中的核心，奠定全书基调。

### 结构化输入（表单字段）

| 字段分组 | 字段名 | 说明 | 是否必填 |
|----------|--------|------|----------|
| 基础信息 | 题材 | 如：仙侠、都市、科幻、现实主义等 | 推荐填写 |
| 基础信息 | 目标读者 | 如：男频、女频、全年龄等 | 可选 |
| 基础信息 | 视角 | 第一人称 / 第三人称有限 / 上帝视角 | 推荐填写 |
| 基础信息 | 主人公基础特质 | 性别、年龄段、核心性格标签等 | 推荐填写 |
| 工程指标 | 单章目标字数 | 如：3000字 / 5000字 | 可选 |
| 工程指标 | 总目标章节数 | 如：200章 | 可选 |
| 工程指标 | 总目标字数 | 如：60万字 | 可选 |
| 工程指标 | 内容分配计划 | 节奏模型，如：开篇20%铺垫 → 30%发展 → 30%高潮 → 20%收束 | 可选 |
| 核心灵魂 | 小说功能 | 治愈 / 解压 / 感官刺激 / 深度思考 等 | 推荐填写 |
| 核心灵魂 | 核心大纲 | 主要讲了一个什么故事（骨架初稿，可不完善）| 推荐填写 |
| 核心灵魂 | 基础世界观 | 故事发生的基本背景设定 | 可选 |

**注意**：所有字段均可留空，AI 会根据已填内容进行脑补补全。

### AI 生成任务
- 输入：用户填写的结构化字段。
- 处理：分析各字段间的逻辑关系，脑补缺失维度，推演特征冲突。
- 输出：**数千字**的纯文本《创世书》，涵盖小说各维度的特征描述。

### 用户交互
- 生成后可「重新生成」。
- 生成后可在文本编辑器中直接修改调整。
- 确认后点击「完成创世纪」进入下一阶段。

### 产出物
**《创世书》**：全书最高宪法，所有后续纪元的根基依据。

## 纪元二：神世纪 (Theogony) —— 世界观法则构建

### 目标
完成世界观的全面构建，如同神明塑造世界。包括力量体系、地理、政治、命名体系等。

### 结构化输入（动态字段）

与创世纪不同，**神世纪的字段不是固定的**，而是：

1. **AI 根据《创世书》内容自动生成字段名和预填内容**。
   - 例如：仙侠题材 → 生成「境界体系」「宗门/流派」「灵力/法宝」等字段。
   - 例如：现实主义 → 不会生成力量体系字段，而可能生成「社会阶层」「城市地理」等。
2. **用户可以修改 AI 预填的内容**。
3. **用户可以增加或删除字段**。
4. **提供一个通用「备注」字段**，用户可以备注任何补充信息。

### 命名库规划
神世纪还承担一个重要职责：**为故事中的场景、地名、人物等规划命名风格和具体名称**。这属于世界观构建的一部分，体现在动态字段中。

### AI 生成任务
- 输入：用户修改后的结构化字段 + 《创世书》。
- 处理：将零散字段扩充为逻辑自洽的宏大叙述。
- 输出：**数千至上万字**的纯文本《神世法》，将整个世界体系娓娓道来，涵盖政治、地理、力量、禁忌、文化、命名等全部维度。

### 用户交互
- 同创世纪：可重新生成、可手动编辑、确认后进入下一阶段。

### 产出物
**《神世法》**：正文续写时的背景逻辑审查依据。

---

## 纪元三：时空纪 (Chronos) —— 完整剧情构建

### 目标
构建小说的完整剧情时间线。明确全部剧情走向：时间线、关键事件、高潮、转折、结局。

### 结构化输入（可视化时间轴）

1. **AI 根据《创世书》+《神世法》自动生成初始时间线**。
2. 时间线由多个**节点**组成，每个节点字段：
   - **发生时间**：可为空（某些事件无需精确时间）。
   - **事件描述**：该节点发生了什么。
3. **用户可以修改、添加、删除时间节点**。
4. **提供一个通用「备注」字段**，可备注任何补充信息。

### AI 生成任务
- 输入：用户修改后的时间轴 + 《创世书》+ 《神世法》。
- 处理：
  1. 补全节点间的逻辑过渡。
  2. 生成详细剧情叙述。
  3. **额外输出：剧情剪辑建议**——叙事起点是否从时间线开头开始？是否使用倒叙/插叙？具体锚点在哪？
- 输出：**上万字**的纯文本《时空律》，包含完整剧情叙述 + 叙事剪辑方案。

### 用户交互
- 同前：可重新生成、可手动编辑、确认后进入下一阶段。

### 产出物
**《时空律》**：全书剧情的上帝视角蓝图 + 叙事结构指导。

---

## 纪元四：人世纪 (Anthropocene) —— 人物卡构建

### 目标
推演并定义所有人物的完整人物卡。

### 结构化输入（人物卡阵列）

1. **AI 根据前序产出物（《创世书》+《神世法》+《时空律》）自动生成人物卡**。
2. 每张人物卡字段：

| 字段名 | 说明 |
|--------|------|
| 姓名 | 人物名称 |
| 归属势力 | 所属阵营/组织/家族 |
| 核心动机 | 欲望与恐惧，驱动行为的底层逻辑 |
| 性格 | 性格底色与主要特征 |
| 习惯 | 日常行为习惯 |
| 行为/语言癖好 | 口癖、小动作等辨识度特征 |
| 关系 | 与其他人物的关系网络 |

3. **用户可以修改、添加、删除人物卡**。

### AI 生成任务
- 输入：用户修改后的人物卡 + 全部前序产出物。
- 处理：为每个人物推演行为模式，生成详细人物小传。
- 输出：**上万字**的纯文本《众生相》，包含每个人物的详细描述。

### 用户交互
- 同前：可重新生成、可手动编辑、确认后进入下一阶段。

### 产出物
**《众生相》**：续写时对话风格与行为一致性的核心来源。

## 纪元五：因果一纪 (Causality I) —— 每章大纲构建

### 目标
将《时空律》的宏观剧情拆解为每一章的逻辑指令。

### 构建流程（两级生成）

#### 第一级：章节框架生成
1. **AI 生成**每章的：
   - **章节名称**
   - **1-2 句话核心剧情描述**
2. **用户干预**：调整顺序、修改名称/描述、增删章节。

#### 第二级：章节详细描述生成
3. **AI 根据定稿后的章节框架**，为每章生成约 **500 字**的详细逻辑描述：
   - 这一章发生了什么？
   - 必须达成的叙事结果是什么？
   - 与前后章的衔接逻辑。
4. **用户可对每章详细描述进行编辑调整**。

### 用户交互
- 两级分别可重新生成。
- 确认后进入下一阶段。

### 产出物
**《因果录》**：章节级的执行清单。每章包含名称 + 简短描述 + 详细描述。

---

## 纪元六：因果二纪 (Causality II) —— 分镜构建

### 目标
将每章的剧情转化为详细的分镜脚本，作为正文写作的直接提示源。

### 特殊机制
因果二纪与前五个纪元有一个关键区别：

> **分镜可以按需生成**。全书分镜数量庞大，在 Planning 阶段不必一次性完成全部分镜。  
> 用户可以在 Planning 阶段跳过此纪元或只完成部分章节的分镜。  
> 但在 Writing Workspace 中**写某章正文之前，必须先完成该章的分镜生成**。

### 结构化输入（分镜卡阵列）

每个分镜（镜头）的字段：

| 字段名 | 说明 |
|--------|------|
| 镜头序号 | 该章内的顺序编号 |
| 登场人物 | 该镜头涉及的人物 |
| 场景/地点 | 该镜头发生的场景锚点 |
| 事件 | 该镜头的核心事件 |
| 动作细节 | 人物的具体动作 |
| 心理/情绪状态 | 人物内心活动与情绪走向 |

### 构建流程
1. **AI 为指定章节生成分镜卡阵列**（基于《因果录》中该章的详细描述 + 全部前序产出物）。
2. **用户修改、增删分镜**。
3. **AI 将结构化分镜卡转化为非结构化的详细分镜描述文本**。
4. 用户可编辑调整。

### 产出物
**《命理镜》**：正文写作的直接提示源。按章节组织，每章包含完整的分镜描述。
