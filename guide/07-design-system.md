# 设计系统 — 视觉风格与设计语言

## 一、设计定位：「白玉圣洁」

纪元的设计语言融合两个基因：

- **白玉极简**：以近乎纯白为基底，大面积留白、精确排版，呈现如白玉般温润圣洁的质感。
- **神性空灵美学**：从"创世纪""神世法""时空律"等命名哲学中提取仪式感与超越感。

最终呈现：**一个圣洁的、有呼吸感的创作圣殿**——白玉为体，金光点睛，如同光照入殿堂。

---

## 二、色彩体系

### 主色调：白玉素净

以纯净白灰为骨架，营造温润如玉的视觉底色。

| 语义 | 色值 | CSS 变量 | 用途 |
|------|------|----------|------|
| 背景·白玉 | `#FCFCFC` | `--background` | 主背景 |
| 背景·卡片 | `#FFFFFF` | `--card` | 卡片、面板背景 |
| 背景·次层 | `#F4F4F5` (zinc-100) | `--secondary` | 侧边栏高亮、hover 态背景 |
| 文字·主体 | `#18181B` (zinc-900) | `--foreground` | 正文文字 |
| 文字·次要 | `#71717A` (zinc-500) | `--muted-foreground` | 辅助文字、标签 |
| 边框·玉纹 | `#E4E4E7` (zinc-200) | `--border` | 分割线、边框 |

### 神性点睛色：「灵光」

在白色基底上，用更深沉的古金色制造"神性感"。白底上金色更内敛、更有质感。

| 语义 | 色值 | 用途 |
|------|------|------|
| 灵光·金 | `#B4823E` → `#D4A574` | 纪元标题、完成状态、产出物名称（如"《创世书》"） |
| 灵光·银 | `#9CA3AF` → `#D1D5DB` | AI 生成中的光标闪烁、加载指示 |
| 灵光·辉 | `rgba(180, 130, 70, 0.10)` | 极淡金色渐变背景，用于当前激活纪元的面板 |

### 使用原则

- **金色使用率 < 5%**：仅用于纪元名称、产出物标题、关键状态指示。绝不大面积铺色。
- **白灰占 95%**：保持白玉般的克制与高级感。
- **禁用饱和色**：不使用纯蓝、纯红、纯绿等高饱和色彩。错误状态用柔红，成功状态用灵光金。
- **默认亮色模式**：产品以亮色模式为默认，白玉圣洁感需要光亮底色来承载。

## 三、排版体系

### 字体

| 用途 | 字体栈 | 说明 |
|------|--------|------|
| 界面文字 | `Inter, -apple-system, sans-serif` | 清晰锐利的现代无衬线体 |
| 纪元标题/产出物名称 | `Noto Serif SC, serif` | 宋体/衬线体，赋予神性仪式感 |
| 正文写作区 | `Noto Serif SC, Georgia, serif` | 衬线体，阅读舒适，有文学质感 |
| 代码/结构化字段 | `JetBrains Mono, monospace` | 等宽字体 |

### 字号层级

| 层级 | 大小 | 用途 |
|------|------|------|
| Display | 36-48px | Landing 页标题、纪元大标题 |
| H1 | 28-32px | 页面标题 |
| H2 | 22-24px | 区块标题 |
| H3 | 18-20px | 卡片标题、字段分组标题 |
| Body | 15-16px | 正文、表单内容 |
| Caption | 13-14px | 辅助说明、时间戳 |
| Micro | 11-12px | 标签、徽章 |

### 行距与间距

- 正文行距：1.75（写作区为 1.8-2.0，宽松舒适）。
- 段间距：与行距等高或 1.5 倍行距。
- 组件间距遵循 **4px 基础网格**（4, 8, 12, 16, 24, 32, 48, 64）。

## 四、神性设计语言

### 4.1 「仪式感」过渡

纪元之间的切换不是简单的页面跳转，而是带有仪式感的过渡：

- **纪元完成动画**：点击"完成 XX 纪元"后，页面内容淡出，纪元名称以衬线大字居中浮现（如"创世已成"），停留 1-2 秒后淡入下一纪元。
- **过渡文案示例**：
  - 创世纪完成 → "创世已成，万物有法"
  - 神世纪完成 → "诸法已定，世界成形"
  - 时空纪完成 → "时空已编，因果可循"
  - 人世纪完成 → "众生已塑，命运待书"
  - 因果一纪完成 → "因果已录，篇章待启"
  - 因果二纪完成 → "分镜已成，执笔可书"

### 4.2 「呼吸感」动效

- **AI 生成状态**：不使用常规 spinner。使用一道极淡的金色光晕从文字末端缓缓脉动，如同文字在"呼吸"。
- **悬浮态**：卡片/按钮 hover 时，边框泛起极淡的金色辉光（`box-shadow: 0 0 24px rgba(180, 130, 70, 0.14)`）。
- **所有动画时长**：200-400ms，缓动函数 `ease-out`。不急不缓，保持沉静。

### 4.3 「卷轴感」布局

- **纪元页面**：内容以纵向单轴滚动展开，如同展开一卷古卷。避免多 tab 切换打断阅读流。
- **产出物展示**：《创世书》《神世法》等文本展示时，使用窄幅居中布局（max-width: 720px），左右大面积留白，模拟书卷阅读体验。
- **章节列表**：纵向排列，每章之间有细微的分割线，如同竹简的缝隙。

### 4.4 「圣殿感」空间

- **大面积留白**：页面不拥挤。内容区域占据视口 60-70%，两侧留白。
- **Writing Workspace 三栏**：中轴创作区最宽（50-60%），左右翼各占 20-25%，可折叠。
- **背景纹理**：可选——极淡的噪点纹理或细微的径向渐变，增加白玉温润质感。不能喧宾夺主。

## 五、组件风格指南

### 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| 主要操作 | 墨色底 + 白色文字，hover 微光 | "AI 创世""完成纪元"等核心操作 |
| 次要操作 | 灰色文字 + 玉色边框 | "重新生成""返回"等辅助操作 |
| 神性操作 | 金色文字 + 金色边框（极细），hover 金色辉光 | 特别重要的里程碑操作（极少使用） |
| Ghost | 无边框，hover 显示底色 | 工具栏图标按钮 |

### 卡片

- 背景：`白色`（`bg-card`），1px 玉色边框（`border-border`）。
- 圆角：`8px`（不要过大，保持锐利感）。
- 阴影：极淡阴影增加层级感，hover 时可增强。
- Hover：边框泛起极淡金色，可选金色辉光。

### 表单输入

- 背景：白色或透明。
- 边框：玉色（`border-border`），focus 态加深。
- 文字：墨色（`text-foreground`）。
- Placeholder：灰色（`text-muted-foreground`）。
- 输入框高度：40-44px。

### 下拉框 / 选择器

- 与表单风格一致。
- 弹出面板：白色背景，玉色边框。
- 选中项：微灰背景高亮（`bg-secondary`）。

### 分割线

- 颜色：玉色（`border-border`）。极细（1px）。
- 用于区块分隔，不要过多使用。

## 六、图标与图形

- **图标库**：Lucide React（线性风格，与极简美学契合）。
- **图标粗细**：strokeWidth 1.5（比默认的 2 更精致）。
- **纪元图标**：每个纪元可使用一个具有象征意义的符号图标：
  - 创世纪：✦（星/原点）
  - 神世纪：◈（棱镜/法则）
  - 时空纪：◉（时间之环）
  - 人世纪：◎（众生之眼）
  - 因果一纪：⟐（因果之链）
  - 因果二纪：▣（分镜之格）
- 这些符号可直接使用 Unicode 字符或自定义 SVG，保持线性风格。

## 七、响应式策略

| 断点 | 布局 |
|------|------|
| Desktop ≥1280px | 完整三栏（Writing）/ 宽幅表单（Planning） |
| Tablet 768-1279px | 双栏（Writing 侧栏可折叠）/ 标准表单 |
| Mobile <768px | 单栏，侧栏转为底部抽屉。写作体验优先级低于 Planning |

**注意**：本产品核心场景是桌面端深度创作，移动端为辅助查看。

## 八、设计关键词速查

供开发时快速对齐审美标准：

> **圣洁 · 克制 · 空灵 · 仪式感 · 大留白 · 金光点睛 · 衬线文学感 · 纵轴卷轴流 · 白玉温润**

**禁忌**：
- ❌ 圆润可爱风（大圆角、彩色气泡）
- ❌ 高饱和色彩
- ❌ 密集的 Dashboard 风格（表格堆叠）
- ❌ 花哨动画（弹跳、旋转）
- ❌ 深色模式作为默认
